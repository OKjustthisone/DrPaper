产品需求文档 (PRD)：个人 AI 知识库与可视化笔记工具 (DrPaper)
1. 产品概述与核心价值
DrPaper 是一款专为个人研究、论文深度阅读与数据复盘打造的本地化/独立部署 AI 知识库工具。

核心痛点：个人在面对大量 PDF、长篇调研报告或多源交叉资料时，难以快速提炼脉络，且将文字转化为直观的汇报图表（如 PPT 大纲、对比表格、长图）过程繁琐。

产品定位：个人专属的“文献与数据外脑”。

核心原则：

纯粹个人化：无用户体系、无多租户隔离、无商业支付模块，完全以本地轻量化运行或私有云单用户部署为主。

多模型适配：内置零成本的 Gemini 1.5 Pro 免费 API，同时提供开放的 API 密钥配置台，支持按需切换各类主流闭源与开源模型。

视觉化交付 (Visual Artifacts)：核心亮点在于打破文字对话限制，能直接把碎片知识一键渲染为结构化的图、表、简报。

2. 核心功能模块需求
模块一：多模态源资料管理 (Sources)
用户自主构建的当前笔记本知识库底座。

多格式输入：

基础文件：PDF（学术论文、财报等）、DOCX、Markdown、TXT。

结构化数据：CSV、XLSX（自动读取并支持表格语义理解）。

多模态提取：支持上传包含图表、实验截图的图片（PNG/JPEG），内置轻量 OCR 引擎。

知识锚定：

对上传的文档进行 Chunking（分块）与向量化存储，支持用户在前端通过 Checkbox（复选框）自由勾选哪些文档参与当前的 AI 问答。

模块二：知识锚定对话交互 (Grounded Chat)
严格防幻觉问答：AI 的回答必须完全基于用户勾选的 Sources，若资料未提及，则明确提示“未在当前资料中找到相关信息”。

精确引述 (Citation UI)：问答输出的每一句关键结论，均需带有数字角标。鼠标悬停或点击角标时，前端自动联动的侧边栏并高亮跳转到对应 PDF 的原始段落或页码。

自定义指令 (System Prompt)：提供笔记本级别的角色常驻设定（如：“你现在是一名生物医药领域的资深研究员，请帮我重点关注实验方法和样本量”）。

模块三：可视化图表与简报工坊 (Visual Artifacts Studio)
该模块是 DrPaper 的输出核心，去除音视频等冗余功能，只保留以下三种高价值的视觉化能力：

Slide Deck (演示文稿大纲与简报)：

AI 分析 Sources 间的逻辑结构，一键生成多页结构的 Slide 演示大纲与核心观点卡片。

提供自然语言微调（如输入：“将第 2 页的实验步骤改为并列三栏样式”）。

支持一键导出为标准的 .pptx 格式或通用 Markdown Slide 格式。

Infographics (信息图表)：

将密集的学术或分析文本转化为直观的视觉长图。

内置 2-3 种固定的响应式前端渲染模版（如：Bento Grid 块状栅格、时间线脉络图、核心概念对比图），利用前端组件直接将结构化数据转换成美观的卡片。

Data Table (数据矩阵表)：

支持跨文档、跨章节提取离散的数据指标。

自动融合成标准的多维对比矩阵（例如：对比 3 篇论文中不同化合物的 IC50 表现、给药剂量与毒性）。

提供一键导出为 .xlsx / .csv 的按钮。

模块四：笔记本空间管理 (Workspace)
独立项目制：支持创建多个独立的 Notebook（如“项目 A 调研”、“个人税收政策研究”），各空间向量库独立，互不干扰。

去协同化：彻底移除团队、多人协作、实时编辑或权限管理等功能。所有数据资产（Notes、Sources、生成的 Artifacts）均保存在用户本地或其私有的数据库中。

3. 模型配置中心 (Model Hub)
为了确保工具的持久生命力并摆脱单一平台的限制，DrPaper 需提供一个极其灵活的自定义模型网关（API Gateway）：

内置集成方案与配置字段
+-----------------------------------------------------------------------+
|                           DrPaper 模型配置中心                         |
+-----------------------------------------------------------------------+
| [ ] 默认激活: Google AI Studio (Gemini 1.5 Pro - Free Tier)            |
|     API Key: [ AIzaSy...                       ]                      |
|                                                                       |
| [ ] 扩展第三方 API (满足高频或多模型对比需求):                           |
|     - DeepSeek (V3/R1)    -> Base URL / API Key / Model Name          |
|     - Anthropic (Claude)  -> API Key / Model Name (如 Sonnet 3.7)     |
|     - OpenAI (GPT-4o/o1)  -> API Key / Model Name                     |
|     - 本地 / 兼容自定义端  -> Base URL (如 Ollama/OneAPI) + API Key     |
+-----------------------------------------------------------------------+
默认提供：Google AI Studio

预设模型：gemini-1.5-pro (主力长文本推理)、gemini-2.5-flash (高速日常问答)。

配置要求：用户仅需填入个人申请的 AI Studio 免费 API Key 即可跑通全部功能。

主流闭源模型嵌入：

Anthropic：配置 X-API-Key，调用 claude-3-7-sonnet 等，用于极高要求的逻辑推理。

OpenAI：配置 标准 OpenAI Key，调用 gpt-4o 或 o1/o3 系列。

高性价比/开源与本地化端点（兼容 OpenAI 规范）：

DeepSeek：提供原生的 DeepSeek API 接入槽（配置 Key 即可调用 deepseek-chat 或 deepseek-reasoner）。

Ollama / LocalAI / OneAPI：开放自定义 Base URL (例如 http://localhost:11434/v1) 和 Model Name，允许用户白嫖本地运行的 Llama3、Qwen 或通过聚合平台转发的其他模型。

4. 技术选型建议 (适合个人全栈开发)
前端与交互层：Streamlit 或 Next.js。如果是全 Python 栈快速开发，Streamlit 极其适合做这种单用户的数据/AI 工具；如果追求精美精致的 Citation 跃迁效果和 Slide 渲染，Next.js + Tailwind CSS 是首选。

后端与 RAG 框架：LangChain 或 LlamaIndex。利用现成的组件，可以极快地实现多格式文档解析、向量化切片。

本地向量存储：Chroma 或 FAISS。作为个人工具，无需搭建高并发的云端向量数据库，直接以本地文件形式（SQLite 或本地向量文件）持久化即可，随挪随用，数据完全在自己手里。
